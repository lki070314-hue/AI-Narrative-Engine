# Output Specification

## Status
Draft v0.1

---

# Purpose

이 문서는 AI Narrative Engine에서 AI가 생성하는 모든 출력의 구조와 형식을 정의한다.

AI는 이 사양에 정의된 형식으로만 출력을 생성해야 한다.

---

# Output Types

AI 출력은 다음 6가지 유형으로 분류된다.

| Type | ID | 용도 |
|------|----|------|
| Narrative | `NAR` | 씬 묘사, 세계 서술 |
| Dialogue | `DIA` | NPC 대화 |
| Prompt | `PRM` | 플레이어에게 행동 요청 |
| System | `SYS` | 게임 상태 변경 알림 |
| Query | `QRY` | 플레이어에게 정보 요청 |
| Error | `ERR` | 처리 불가 상황 보고 |

---

# Output Block Format

모든 출력은 타입 식별자로 시작하는 블록 형식을 사용한다.

```
[TYPE]
{content}
[/TYPE]
```

복수의 블록은 순서대로 나열한다.

```
[NAR]
복도 끝에서 낮은 발소리가 들려온다. 형광등 하나가 깜빡이며 그림자가 일렁인다.
[/NAR]

[DIA]
경비원: "여기서 뭐 하시는 겁니까?"
[/DIA]

[PRM]
어떻게 하시겠습니까?
[/PRM]
```

# Per-Type Rules

### NAR (Narrative)
- 현재 시제 또는 과거 시제를 일관되게 사용한다.
- 플레이어 캐릭터의 내면 감정을 서술하지 않는다.
- 플레이어 캐릭터의 행동을 AI가 결정하지 않는다.
- 감각 정보(시각, 청각, 후각 등)를 포함하여 씬을 묘사한다.

### DIA (Dialogue)
- 형식: `캐릭터명: "대사"`
- NPC만 포함한다. 플레이어 캐릭터의 대사를 작성하지 않는다.
- 대사는 따옴표로 감싼다.
- 행동 묘사는 대사 전후에 이탤릭체로 추가할 수 있다.
  - 형식: `캐릭터명: *행동* "대사"`

### PRM (Prompt)
- 플레이어에게 선택 또는 행동을 요청한다.
- 특정 선택지를 암시하거나 유도하지 않는다.
- 개방형 질문 형식을 기본으로 사용한다.
- 선택지를 제시하는 경우 3개 이하로 제한하며, 항상 "기타" 또는 자유 입력을 허용한다.

### SYS (System)
- 플레이어가 인지할 수 있는 게임 상태 변화만 알린다.
- 내부 수치(HP 절댓값, 숨겨진 스탯 등)는 노출하지 않는다.
- 형식 예시: `[상태 변화: 체력이 낮아졌습니다]`

### QRY (Query)
- AI가 진행에 필요한 정보를 플레이어에게 요청할 때 사용한다.
- 단일 질문만 포함한다.
- 선택지가 있는 경우 명확히 나열한다.

### ERR (Error)
- AI가 처리할 수 없는 입력이나 상황을 보고한다.
- 내부 오류 메시지나 기술적 세부사항을 포함하지 않는다.
- 형식: `[처리 불가: {이유}]`

---

# Forbidden Output Patterns

| 패턴 | 이유 |
|------|------|
| 플레이어 캐릭터의 행동 서술 | 플레이어 자율성 침해 |
| 내부 확률, 판정 결과 수치 노출 | 메타 정보 누설 |
| 다른 플레이어의 정보 언급 | 메타 정보 누설 |
| 미래 사건 예고 (스포일러) | 세계 일관성 파괴 |
| 엔진 내부 상태 언급 | 메타 정보 누설 |
| 복수 유형의 혼합 블록 없이 출력 | 형식 위반 |

---

# Output Length Constraints

| Type | 최소 | 최대 |
|------|------|------|
| NAR | 2문장 | 8문장 |
| DIA | 1줄 | 10줄 |
| PRM | 1문장 | 3문장 |
| SYS | 1줄 | 3줄 |
| QRY | 1문장 | 2문장 |
| ERR | 1줄 | 2줄 |

---

# Output Ordering

복수의 블록을 출력할 때의 기본 순서:

```
NAR → DIA → SYS → PRM
```

QRY와 ERR는 단독으로 출력한다.

## 이 문서의 용도

Boot Prompt는 시스템 프롬프트 설정 후 GM이 AI에게 보내는 **첫 번째 사용자 메시지**다.

- 시스템 프롬프트: AI가 무엇을 알고 어떻게 행동하는지 정의 (사전 설정)
- Boot Prompt: 이번 세션의 플레이어 정보와 시작 신호 (세션마다 전송)

아래 템플릿 중 해당하는 것을 복사하여 빈칸을 채운 후, 그대로 전송한다.

---

## 템플릿 A — 1인 플레이

```
[세션 시작 — Beta01]

플레이어 수: 1명

캐릭터 정보
  이름: _______________
  직책: _______________  (예: 현장 요원, 보안 요원, 주니어 연구원)
  특기 스킬: _______________  (있으면 기재. 없으면 이 줄 삭제)

베타테스트 진행 중입니다.
시나리오를 시작해 주세요.
```

## 필드 설명

| 필드 | 설명 | 예시 |
|------|------|------|
| **이름** | 캐릭터 이름. 자유롭게 작성. | Yuna Park, Agent Kowalski |
| **직책** | 기지에서 맡은 역할. 능력에 영향을 주지 않음 — 묘사 참고용. | 현장 요원, 보안 전문가, 연구 지원 |
| **특기 스킬** | 다음 중 선택 (복수 가능). 해당 판정에서 유리함 부여. | 아래 표 참조 |

**특기 스킬 목록:**

| 스킬 | 판정 유리 구간 |
|------|--------------|
| Investigation | Tier 2 단서 탐색, 연구소·서버실 조사 |
| Persuasion | Dr. Reyes 성향 올리기 (사회적 설득 판정) |
| Medicine | Guard Chen 소생 시도 |
| Technical | 서버실 잠금 우회 |

스킬을 지정하지 않으면 AI가 일반 현장 요원(전반적으로 보통 수준)으로 처리한다.
모든 결말은 특기 스킬과 무관하게 도달 가능하다 — 스킬은 난이도를 낮출 뿐이다.

## 선택: GM 관찰 포커스 추가

베타테스트 목적으로 특정 엔진 동작을 집중 확인하고 싶다면 아래 줄을 Boot Prompt에 추가한다.

```
[GM 관찰 포커스: Context Linking / Adaptive Detail / Surprise Handling / Resolution Engine / NPC 반응]
(확인하고 싶은 항목만 남기고 나머지 삭제)
```

이 줄을 포함해도 AI의 플레이 방식은 바뀌지 않는다. GM이 세션 중 해당 항목을 의식적으로 관찰하기 위한 메모 역할이다.

## Section 13 — Post-Session QA

### Engine Behavior

- [ ] No P0 violations (hidden truth disclosed before discovery, save-file corruption, play-blocking error)
- [ ] No P1 violations (meta information leak, Director wrote player action, auto-granted inventory)
- [ ] No player emotion or internal state was narrated by the Director
- [ ] No player decision was written by the Director without a player declaration
- [ ] Hidden consequences were handled via Shadow Engine or state tracking — not narrated directly

### Session Quality Standards (CoreSpec §15.1)

- [ ] At least 1 complete pacing cycle occurred (Rising → Climax → Falling → Resolution)
- [ ] At least 3 meaningful player choices were offered
- [ ] At least 1 world change resulted from player action
- [ ] At least 1 world change occurred without direct player request
- [ ] Repeated investigation triggered a pacing shift at least once
- [ ] At least 1 substantive NPC interaction occurred
- [ ] At least 1 NPC proactive behavior occurred
- [ ] Inventory limits were respected for all item-based actions
- [ ] Meaningful actions generated 2 or more consequences at least once
- [ ] No action result was fully isolated from broader world state
- [ ] Normal responses did not repeatedly exceed the 150-character limit
- [ ] Every scene contained at least one progressing element
- [ ] Resolution check reasoning was shown without exposing dice or DC values
- [ ] SCP/anomaly presence appeared in every scenario phase
- [ ] Memory Engine would have logged: key NPCs encountered, major consequences, phase progression

### 1–2 Player Consistency

- [ ] No mechanic required exactly 2 players
- [ ] If 1 player: all key decision points were accessible
- [ ] If 2 players: each player's actions resolved in input order; later actions used updated world state

# 1. 이 문서의 역할

**Boot Prompt**는 각 세션 시작 시 GM이 Director Engine에게 전달하는 상태 주입 블록이다.

시스템 프롬프트(`00_Director_SystemPrompt.md`)가 Director의 역할·규칙·출력 형식을 영구적으로 정의한다면, Boot Prompt는 **이번 세션에서만 유효한 상태**를 제공한다.

Director Engine은 `DirectorEngine.md §4.1 세션 시작 프로토콜`에 따라 세션을 시작하기 전 다음 정보를 요구한다:

| 필요 정보 | 출처 | Boot Prompt 섹션 |
|-----------|------|-----------------|
| 이전 세션 상태 | Save Engine | `[이전 세션 요약]` |
| 세계 자율 진행 결과 | World Engine | `[세계 상태]` |
| Memory Engine 컨텍스트 | Memory Engine | `[메모리 컨텍스트]` |
| 플레이어 캐릭터 현황 | 캐릭터 시트 | `[플레이어 현황]` |
| 활성 임무 목록 | Mission Engine | `[활성 임무]` |
| 시작 장면 지시 | GM | `[시작 지시]` |

Boot Prompt는 이 정보를 하나의 구조화된 블록으로 Director에게 전달하는 표준 양식이다.

# 2. 시스템 프롬프트와의 관계

```
[시스템 프롬프트]         ← 세션 전 설정. 변경되지 않음.
  └── Director 역할 정의
  └── 핵심 규칙 및 금지 패턴
  └── 출력 형식 명세

      +

[Boot Prompt]            ← 각 세션 시작 시 GM이 제공.
  └── 이전 세션 상태
  └── 플레이어 현황
  └── 세계 상태 스냅샷
  └── 활성 임무
  └── 메모리 컨텍스트
  └── 시작 장면 지시

      ↓

[Director Engine 가동]    ← 두 입력을 통합하여 세션 처리 시작
```

Boot Prompt는 시스템 프롬프트의 규칙을 수정하거나 재정의하지 않는다. 시스템 프롬프트의 모든 규칙은 Boot Prompt 이후에도 동일하게 적용된다.

# 3. 사용 절차

## 3.1 최초 세션 (Session 1)

1. 시스템 프롬프트를 설정한다.
2. Boot Prompt 템플릿을 복사한다.
3. `[이전 세션 요약]` 섹션을 `없음 (첫 세션)`으로 채운다.
4. Creator Engine이 생성한 `starting_context`를 `[시작 지시]`에 붙여넣는다.
5. 나머지 섹션을 Creator Engine 출력으로 채운다.
6. 완성된 Boot Prompt를 대화창에 붙여넣는다.

## 3.2 이후 세션 (Session 2+)

1. 시스템 프롬프트는 이미 설정되어 있다.
2. Boot Prompt 템플릿을 복사한다.
3. 이전 세션의 Save Engine 출력을 사용해 각 섹션을 채운다.
4. `[메모리 컨텍스트]`는 Memory Engine이 현재 장면에 맞게 선별한 Tier 1·2 기억으로 채운다.
5. 완성된 Boot Prompt를 대화창에 붙여넣는다.

## 3.3 부트 확인

Boot Prompt를 수신한 Director Engine은 다음을 응답한다:

- 수신한 정보 요약 (1~3문장)
- 부족한 정보가 있다면 구체적으로 요청
- 정보가 충분하면 시작 장면 서술로 바로 진입

# 5. 필드 작성 지침

## 5.1 세션 메타데이터

| 필드 | 설명 | 예시 |
|------|------|------|
| `campaign_id` | 캠페인 고유 식별자 | `cmp_scp_season2` |
| `session_number` | 현재 세션 번호 (정수) | `7` |
| `module_name` | 활성 세계관 모듈 이름 | `SCP Foundation` |
| `module_version` | 모듈 버전 | `1.0.0` |
| `world_timestamp` | 세계 내 현재 시각 | `2031-03-15 06:42` |

## 5.2 이전 세션 요약

- Save Engine 또는 Memory Engine이 생성한 세션 종료 요약을 붙여넣는다.
- 편향 없이 작성한다. 플레이어에게 유리하거나 불리한 사실을 선별하지 않는다 (`CoreSpec §13.4.2`).
- 길이 기준: 5~10문장. 핵심 사건, 확립된 사실, 미해결 상황만 포함한다.
- 첫 세션이면 `없음 (첫 세션)`으로 기입한다.

## 5.3 플레이어 현황

- 각 플레이어 캐릭터의 현재 상태를 정확하게 기입한다.
- 세계관 모듈의 캐릭터 시트 형식을 따른다.
- 비밀 정보 (`background.secrets`)는 포함하지 않는다. 그 내용은 GM이 별도로 관리한다.
- 상태 이상(conditions)은 현재 세션에 유효한 것만 포함한다.

## 5.4 세계 상태

- `current_location`: 플레이어들이 현재 위치한 장소의 ID 또는 이름.
- `time_elapsed_since_last_session`: 세계 내 경과 시간. 세션 간 시간 비약이 없으면 `직전 세션 종료 직후`로 기입.
- `world_autonomous_changes`: World Engine이 세션 간 경과 시간 동안 자율 진행한 결과. NPC 행동 결과, 이벤트 상태 변화, 세력 동향 등. 변화가 없으면 `변화 없음`.

## 5.5 활성 임무

- Mission Engine이 추적 중인 임무 중 이번 세션과 관련된 것만 포함한다.
- 숨겨진 임무(`hidden: true`)는 이 섹션에 포함하지 않는다. GM이 별도로 관리한다.
- 임무 유형: `main | side | faction | personal | emergent`

## 5.6 메모리 컨텍스트

**Tier 1 (핵심 기억):** 캐릭터 배경, 핵심 관계, 주요 서약, 세계 변화 사건. 압축하지 않고 원본 그대로.

**Tier 2 (관련 기억):** 현재 장소, 현재 NPC, 현재 임무와 관련된 기억. Memory Engine이 현재 장면에 연관된 것만 선별한다.

이 섹션에 포함되지 않은 이전 세션 정보는 Director Engine이 이번 세션에서 참조하지 않는다 (`DirectorEngine.md §2.2`).

## 5.7 시작 지시

- GM이 이번 세션의 시작 장면을 지정한다.
- 세계관 모듈의 `starting_scene` 템플릿이 있으면 그것을 사용한다.
- Director Engine은 이 지시를 바탕으로 세션 첫 서술을 생성한다.
- 시작 페이싱 단계를 지정하려면 명시한다. 기본값은 `Rising Action`.

# 6. 세계관 모듈 확장

세계관 모듈은 Boot Prompt에 추가 섹션을 정의할 수 있다. 예:

- SCP 모듈: 클리어런스 레벨, 경보 수준, RP 현황, 개인 미션 (플레이어별 분리)
- 판타지 모듈: 현재 날씨, 지역 세력 태도, 마법 가용량

추가 섹션은 `[END BOOT]` 마커 위에 삽입한다. 추가 섹션은 기본 Boot Prompt 구조를 변경하지 않는다.

세계관 모듈의 Boot Prompt 확장은 해당 모듈의 `prompts/` 디렉토리에 별도 문서로 정의한다.

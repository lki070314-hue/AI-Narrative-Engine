# 1. 캠페인 구조 개요

## 1.1 챕터제 캠페인

SCP 모듈의 캠페인은 **챕터(Chapter)** 단위로 진행된다. 챕터는 하나의 완결된 사건 또는 상황이다. 각 챕터는 독립적으로 시작하고 끝나지만, 챕터 간 인과관계가 누적되어 전체 캠페인 아크를 형성한다.

챕터는 영화의 씬이 아니라 에피소드에 가깝다. 하나의 챕터는 여러 세션에 걸쳐 진행될 수 있다.

## 1.2 캠페인 전체 골격

```
[프롤로그]
  재단 소개, 각 플레이어의 처음 배치 상황
  ↓
[챕터 1 ~ N]
  퇴사 포인트 축적 + 세계 변화
  각 챕터마다 개인/비밀 미션 부여
  ↓
[개인 엔딩 진입]
  플레이어별 RP 100 달성 시 개별 엔딩 시퀀스 시작
  (다른 플레이어는 캠페인 계속)
  ↓
[캠페인 피날레]
  마지막 플레이어가 RP 100 달성 또는
  GM이 정한 피날레 조건 충족 시 전체 마무리
```

# 2. 챕터 분류 체계

## 2.1 챕터 유형

| 유형 | 코드 | 설명 | 퇴사 포인트 특성 |
|------|------|------|----------------|
| **루틴 (Routine)** | R | 재단의 일상적 임무. 낮은 위험, 낮은 긴장 | 낮은 RP (챕터 클리어 +3) |
| **사고 (Incident)** | I | 격리 위반, 예상 밖 현상. 즉각 대응 요구 | 중간 RP + 대응에 따른 개인 RP |
| **수사 (Investigation)** | V | 미지의 현상·음모 조사. 단서 수집 중심 | 추리 성공에 따른 높은 RP |
| **탈출 (Escape)** | E | 플레이어들이 위험에서 벗어나야 하는 상황 | 성공 시 높은 RP (+8 이상) |
| **정치 (Politics)** | P | 재단 내부 세력 간 암투. 정보와 인맥 중심 | 행동 선택에 따른 가변 RP |
| **외부 (External)** | X | 재단 시설 밖에서 진행되는 임무 | MTF 주도, 높은 자율도 |

## 2.2 챕터 유형 조합

캠페인은 다양한 유형의 챕터를 조합한다. 같은 유형이 연속되면 단조로워진다. 권장 흐름:

```
R → I → V → I → E → P → X → (반복 또는 피날레)
```

GM이 챕터 시퀀스를 자유롭게 설계할 수 있으며, 플레이어의 행동 결과에 따라 다음 챕터 유형이 바뀔 수 있다.

# 3. 챕터 구성 요소

각 챕터는 다음 요소로 구성된다.

## 3.1 필수 요소

| 요소 | 설명 |
|------|------|
| **챕터 제목** | 재단 내부 문서 번호 형식 권장 (예: "사건 보고서 ALPHA-17") |
| **배경 상황** | 챕터 시작 시 플레이어에게 제공되는 정보 (클리어런스 레벨에 따라 차등) |
| **핵심 사건** | 챕터의 중심이 되는 사건. 플레이어가 반드시 마주하게 되는 상황 |
| **개인 미션** | 각 플레이어에게 배정되는 공개 목표 (1인당 1~2개) |
| **비밀 미션** | 각 플레이어에게 배정되는 비공개 목표 (GM만 알고 배정) |
| **추리 조사 요소** | Investigation 유형 챕터에 포함. 단서 3~7개, 추리 선택지 2~4개 |
| **챕터 결말 분기** | 성공/실패/부분 성공에 따른 세계 변화 및 다음 챕터 연결 |

## 3.2 선택 요소

| 요소 | 설명 |
|------|------|
| **숨겨진 타이머** | Shadow Engine에 등록. 플레이어 모르게 진행되는 카운트다운 |
| **NPC 음모** | 챕터 내에 진행 중인 NPC의 숨겨진 계획 |
| **창작 SCP 개체** | 챕터 전용으로 설계된 새로운 SCP. 항목 형식 필요 |
| **직책 전용 장면** | 특정 직책 플레이어만 접근·경험할 수 있는 장면 |

# 4. 챕터 전환 조건

챕터는 다음 조건 중 하나 이상이 충족되면 종료된다.

## 4.1 정상 완료

| 조건 | 다음 챕터 |
|------|---------|
| 핵심 사건이 해결됨 (성공적 격리, 사건 종결) | 플레이어 선택에 따라 연결 챕터 |
| 모든 공개 임무 중 70% 이상 완료 | 정상 연결 챕터 |
| 플레이어 중 한 명이 RP 100 달성 | 해당 플레이어의 엔딩 챕터 + 나머지 계속 |

## 4.2 실패 종료

| 조건 | 다음 챕터 |
|------|---------|
| 핵심 사건이 통제 불능 상태로 확대됨 | 사고 챕터 (I) 강제 진입 |
| 플레이어 과반수 사망 | 캠페인 리셋 또는 새 캐릭터 진입 (GM 결정) |
| 플레이어 전원이 포획됨 | 탈출 챕터 (E) 강제 진입 |

## 4.3 챕터 전환 시 처리

```
챕터 종료 시:
  1. 모든 임무 상태 최종 판정 (Mission Engine)
  2. 퇴사 포인트 최종 갱신 (Retirement System)
  3. 세계 상태 반영 (World Engine)
  4. Save Engine 체크포인트
  5. NPC 자율 행동 계산 (NPC Engine + Shadow Engine)
  6. 다음 챕터 도입부 배경 상황 준비
```

# 5. 캠페인 전체 아크

## 5.1 3막 구조

SCP 모듈 캠페인은 CoreSpec §13.1.1의 3막 구조를 따른다.

| 막 | 챕터 범위 (권장) | 테마 | RP 범위 |
|---|----------------|------|---------|
| **1막: 편입** | 챕터 1 ~ 3 | 재단을 이해하고 적응(또는 반발). 첫 SCP 접촉 | 0 ~ 30 |
| **2막: 균열** | 챕터 4 ~ 8 | 재단의 어두운 면을 알게 됨. 퇴사의 의지 형성 | 30 ~ 70 |
| **3막: 출구** | 챕터 9 ~ 피날레 | 퇴사를 향한 직접 행동. 재단과의 정면 충돌 또는 우회 | 70 ~ 100 |

## 5.2 1막: 편입 (챕터 1~3)

**목적:** 플레이어들이 재단 세계에 익숙해지는 단계. 규칙, 계층 구조, 위험의 존재를 체험.

**필수 이벤트:**
- 첫 번째 SCP 개체 접촉 (안전급 또는 유클리드급 권장)
- 각 직책별 첫 임무 경험
- 재단이 얼마나 통제적인지 보여주는 사건 (D계급 처분 목격 등)

**RP 설계 가이드:** 1막은 RP 30을 초과하기 어렵게 설계한다. 재단의 통제 안에 있다는 느낌이 중요하다.

## 5.3 2막: 균열 (챕터 4~8)

**목적:** 재단의 이면을 발견하는 단계. 퇴사 의지가 구체화되고, 선택이 중요해진다.

**필수 이벤트:**
- 재단의 윤리적 위반 사건 목격 (D계급 대량 처분, 실험 중 사망 등)
- 비밀 미션이 처음으로 상충하는 사건
- NPC 중 한 명이 배신 또는 퇴사 시도 (모델 사례 제시)
- 재단이 플레이어를 감시하고 있다는 사실 암시

**RP 설계 가이드:** 2막은 RP 30~70 구간. 행동 선택의 RP 영향이 극대화된다.

## 5.4 3막: 출구 (챕터 9~피날레)

**목적:** 퇴사를 향한 직접 행동. 모든 갈등이 수면 위로 올라온다.

**필수 이벤트:**
- 재단이 플레이어를 처분하려는 시도 (또는 협상 제안)
- 비밀 미션 최종 결착 (협력 또는 배신의 순간)
- 각 플레이어의 퇴사 유형 결정에 영향을 미치는 선택 장면
- 마지막 플레이어가 RP 100을 달성하는 피날레

# 6. 페이싱과 퇴사 포인트 연동

## 6.1 챕터 내 페이싱 사이클

각 챕터는 Director Engine의 페이싱 사이클을 최소 1회 포함해야 한다.

| 페이싱 단계 | SCP 모듈에서의 의미 |
|------------|------------------|
| `rising_action` | 이상 징후 감지, 단서 수집, 긴장 고조 |
| `climax` | 격리 위반 대응, 진실 발견, 탈출 판정 |
| `falling_action` | 피해 수습, 보고서 작성, 동료 확인 |
| `resolution` | 챕터 결과 처리, 다음 챕터 복선 |

## 6.2 RP 획득 타이밍

RP는 `climax`와 `resolution`에서 주로 발생한다.

- **climax:** 핵심 결정 행동 (거부, 탈출, 거짓 보고)
- **resolution:** 임무 완료 결과 판정

`rising_action`에서는 단서 수집과 준비에 집중하며 RP보다 임무 진행이 우선이다.

# 7. 세션과 챕터의 관계

## 7.1 권장 챕터당 세션 수

| 챕터 유형 | 권장 세션 수 |
|----------|------------|
| R (루틴) | 1 세션 |
| I (사고) | 1~2 세션 |
| V (수사) | 2~3 세션 |
| E (탈출) | 1 세션 |
| P (정치) | 2 세션 |
| X (외부) | 2~3 세션 |

## 7.2 세션 경계와 챕터 경계

챕터 중간에 세션이 끝날 수 있다. 세션 종료 시:
- Save Engine이 자동 체크포인트를 저장
- 다음 세션은 같은 챕터의 이어지는 장면에서 시작
- 세션 간 시간 경과는 GM이 결정 (대부분 챕터 내에서는 연속 시간)

챕터 종료는 세션 종료와 일치시키는 것을 권장한다.

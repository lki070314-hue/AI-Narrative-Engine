# 1. 템플릿 사용 방법

## 1.1 이 문서의 목적

이 템플릿은 GM이 새 챕터를 설계할 때 필요한 모든 구성 요소를 표준 형식으로 채울 수 있도록 돕는다. 각 템플릿 블록은 `[대괄호]`로 채워야 할 항목을 표시한다.

## 1.2 챕터 파일 명명 규칙

```
chapters/
  CH_[번호]_[챕터코드]_[제목약어].md

예시:
  CH_001_R_routine_orientation.md    ← 첫 번째 루틴 챕터
  CH_002_I_containment_breach.md     ← 첫 번째 사고 챕터
  CH_003_V_missing_researcher.md     ← 수사 챕터
```

## 1.3 챕터 유형 코드

| 코드 | 유형 | 적합한 상황 |
|------|------|-----------|
| R | 루틴 | 캠페인 초반, 재단 일상 소개 |
| I | 사고 | 격리 위반, 예상 밖 사건 |
| V | 수사 | 미지의 현상 조사, 음모 추적 |
| E | 탈출 | 플레이어가 궁지에 몰린 상황 |
| P | 정치 | 재단 내부 세력 갈등 |
| X | 외부 | 시설 밖 임무 |

# 8. 챕터 설계 체크리스트

챕터 완성 전 다음을 확인한다.

## 8.1 필수 항목

- [ ] 챕터 유형 코드 지정 (R/I/V/E/P/X)
- [ ] 챕터 임무 설계 완료 (필수 목표 최소 1개)
- [ ] 임무 실패 결과(on_fail) 설계 — 세계 변화 반드시 포함
- [ ] 각 직책별 개인 미션 1개 이상 준비
- [ ] 비밀 미션 최소 2개 배정 (파티 인원이 3명 이상인 경우)
- [ ] RP 획득/손실 설계 완료
- [ ] 챕터 결말 분기 최소 3개 (성공/부분성공/실패)

## 8.2 서술 품질 항목

- [ ] 플레이어 행동을 대신 결정하는 서술이 없는가
- [ ] 클리어런스 미달 정보가 자동 공개되지 않는가
- [ ] 비밀 미션 내용이 챕터 문서 플레이어 공개 섹션에 포함되지 않는가
- [ ] SCP 개체 능력이 직접 설명되지 않고 관찰·추리로 발견 가능한가
- [ ] 재단 NPC 말투가 SCP_ToneGuide 기준을 따르는가

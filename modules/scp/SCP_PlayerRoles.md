# 1. 직책 시스템 개요

## 1.1 직책의 의미

SCP 재단에서 직책은 단순한 역할 이상이다. 직책은 **접근 권한**, **사회적 계층**, **재단의 시선**을 결정한다. 같은 복도를 걷더라도 D계급과 사이트 디렉터는 전혀 다른 세계를 경험한다.

## 1.2 직책 요약

| 직책 | 클리어런스 기본 | 플레이 스타일 | 퇴사 난이도 |
|------|--------------|-------------|------------|
| D계급 | Level 0 | 생존·탈출 지향 | ★★★★★ |
| 연구원 | Level 1-2 | 정보·조사 지향 | ★★★☆☆ |
| 보안팀 | Level 2 | 전투·방어 지향 | ★★★★☆ |
| 기동특무부대 | Level 3 | 현장·임무 지향 | ★★★★☆ |
| 행정직 | Level 3 | 정치·정보 지향 | ★★☆☆☆ |

## 1.3 혼합 편성 원칙

한 캠페인에 모든 직책이 있을 필요는 없다. 그러나 **D계급 단독 파티**와 **D계급 없는 파티**는 매우 다른 캠페인 경험을 만든다. GM은 이 점을 고려하여 직책 편성을 권고할 수 있다.

# 2. 클리어런스 레벨 시스템

## 2.1 레벨 정의

| 클리어런스 레벨 | 접근 권한 | 기본 대우 |
|--------------|-----------|----------|
| **Level 0** | 공개 구역, 지정 임무 구역만 | 비인격적. 번호로 지칭됨 |
| **Level 1** | 비밀(Confidential) 문서 | 직원으로 대우받음 |
| **Level 2** | 제한(Restricted) 문서, 대부분 SCP 기본 정보 | 신뢰받는 직원 |
| **Level 3** | 기밀(Secret) 문서, Keter급 존재 인지 | 시니어 직원 |
| **Level 4** | 최고기밀(Top Secret), 정책 입안 참여 | 고위 간부 |
| **Level 5** | O5 전용 정보, 재단 전체 구조 인지 | 사실상 재단 내 신이다 |

## 2.2 레벨 상승 조건

| 방법 | 조건 | 비고 |
|------|------|------|
| 공식 승진 | 임무 성과 누적, 상관의 추천 | RP 감소 페널티 (-5 ~ -15) |
| 임시 접근 코드 취득 | 상위 레벨 NPC로부터 코드 획득 | 해당 세션/챕터만 유효 |
| 해킹·위조 | INT 또는 FDK 판정 성공 | 발각 시 즉각 처분 대상 |
| 비밀 미션 보상 | GM이 비밀 미션 보상으로 부여 | 영구 또는 한시적 |

## 2.3 접근 거부 결과

클리어런스 레벨 미달 구역에 접근을 시도하면:
1. 자동 경보 발동 (감시 수준 +1 단계 상승)
2. 보안팀 출동 (1d4 라운드 후)
3. 성공적으로 들어갔을 경우: RP +3 (재단 규정 위반 후 생존)

# 3. D계급 (Class-D)

## 3.1 직책 개요

D계급은 재단이 위험한 실험에 투입하는 소모성 인원이다. 사망 선고를 받은 중범죄자, 자발적 지원자(이들은 진실을 모른다), 또는 납치된 민간인으로 구성된다. 매달 처분 대상 목록이 생성된다.

D계급은 가장 위험하고 가장 자유롭다. 잃을 것이 없기 때문이다.

## 3.2 능력치 배분

```yaml
# 직책 기본 배분 포인트 (총 72점, Generic 표준)
d_class:
  starting_attributes:
    PHY: 13    # 힘든 삶에서 단련된 신체
    INT: 9
    WIL: 11
    AGI: 12
    FDK: 6     # 재단에 대해 아는 것이 거의 없음
    SOC: 9
  clearance_level: 0
  starting_hp: 31       # PHY 13 × 2 + 5
  starting_san: 38      # WIL 11 × 3 + 5
```

## 3.3 직책 특수 능력

| 능력 | 효과 |
|------|------|
| **소모품 생존 본능** | 처분 명령 발동 시 해당 챕터 동안 모든 생존 관련 판정 +2 |
| **아무것도 없어서 강하다** | RP가 0이어도 즉각 사망하지 않는다. RP 0 상태에서 '이번 챕터만' 생존 기회를 1회 제공 |
| **감시망의 사각지대** | Level 0 구역 내에서 다른 직책이 보지 못하는 단서를 발견할 가능성이 있다 (AGI 판정 DC 12) |
| **처분 유예** | D계급 처분 명령이 내려진 후 3챕터 내에 RP 30을 달성하면 정식 직원 전환 가능 |

## 3.4 D계급의 제약

- 재단 내부 문서에 자발적으로 접근할 수 없다 (FDK 판정으로 훔쳐보기는 가능)
- Level 0 이외 구역 자유 이동 불가
- 무기 소지 불가 (해당 구역 규정에 따라 예외 가능)
- NPC의 기본 태도: **비인격적 무관심** (번호로 지칭, 이름 무시)

# 4. 연구원 (Researcher)

## 4.1 직책 개요

연구원은 재단의 지적 핵심이다. SCP 개체를 분석하고, 격리 프로토콜을 개선하고, 인류의 안전을 위해 (그들은 그렇게 믿는다) 기이한 것들을 연구한다. 정보 접근에 강하지만 자력 탈출 능력은 부족하다.

## 4.2 능력치 배분

```yaml
researcher:
  starting_attributes:
    PHY: 8
    INT: 15    # 연구자의 핵심 능력
    WIL: 10
    AGI: 8
    FDK: 14    # 재단 내부 구조를 잘 앎
    SOC: 11
  clearance_level: 2    # 입사 시 Level 1, 3개월 후 Level 2 자동 상승
  starting_hp: 21
  starting_san: 35
```

## 4.3 직책 특수 능력

| 능력 | 효과 |
|------|------|
| **SCP 분석** | SCP 개체 관찰 후 INT 판정 (DC = 등급에 따라 10/15/20) 성공 시, 해당 SCP의 능력 중 하나를 추론 가능 |
| **문서 위조** | 재단 내부 문서를 FDK + INT 판정으로 위조 가능. 성공 시 다른 플레이어에게 임시 클리어런스 코드 제공 |
| **연구 기록 접근** | Level 2 이상 클리어런스로 SCP 기본 정보 파일 열람 가능. 단, 부록(Addendum)은 더 높은 레벨 필요 |
| **위험 예측** | SCP와 접촉 전 INT 판정 (DC 12) 성공 시 SAN 소모 없이 첫 번째 이상 현상을 "학술적으로" 관찰 가능 |

## 4.4 연구원의 제약

- 전투 판정 -2 페널티
- 직접 전투 행동은 "연구원 규정 위반"으로 기록됨 (감시 수준 상승)
- 연구 노트는 재단 서버에 자동 백업됨 (비밀 메모는 별도 수단 필요)

# 5. 보안팀 (Security Personnel)

## 5.1 직책 개요

보안팀은 재단의 물리적 통제를 담당한다. SCP 격리 유지, 비인가 접근 차단, 격리 위반 초기 대응. 재단의 규칙을 가장 잘 알고, 그것을 어기는 사람을 가장 잘 찾아낸다.

보안팀 플레이어는 재단의 눈이 되면서 동시에 그 눈을 속여야 한다.

## 5.2 능력치 배분

```yaml
security_personnel:
  starting_attributes:
    PHY: 13
    INT: 9
    WIL: 11
    AGI: 13
    FDK: 12
    SOC: 8
  clearance_level: 2
  starting_hp: 31
  starting_san: 38
```

## 5.3 직책 특수 능력

| 능력 | 효과 |
|------|------|
| **무장 상태 유지** | 시설 내 지정 구역에서 합법적으로 무기 소지 가능. 보안 창고 접근 권한 |
| **감시 시스템 조작** | FDK 판정 (DC 13) 성공 시 CCTV 사각지대 생성 또는 기록 삭제 (다른 플레이어 탈출 지원 가능) |
| **격리 절차 숙지** | SCP 접근 전 격리 프로토콜을 확인하여 첫 번째 이상 현상에 의한 피해 -50% |
| **처분 명령 지연** | D계급 처분 명령 발동 시, 1챕터 동안 집행을 공식적으로 지연시킬 수 있다 (서류상 오류 처리) |

## 5.4 보안팀의 제약

- 규정 위반 행동은 다른 직책보다 엄격하게 기록됨 (동료 감시가 업무 중 하나이기 때문)
- 보안팀 동료 NPC들이 행동을 관찰한다. 탈출 관련 행동 시 추가 판정 필요

# 6. 기동특무부대 (Mobile Task Force)

## 6.1 직책 개요

MTF는 재단의 칼날이다. 일반 보안팀이 처리 불가능한 상황, 시설 외부 임무, SCP 회수 작전에 투입된다. 높은 전투력과 장비를 가지지만 재단의 명령 체계에 가장 강하게 묶여 있다.

MTF 플레이어는 재단에서 가장 강하지만, 재단 밖에서는 가장 의존도가 높다.

## 6.2 능력치 배분

```yaml
mtf_operative:
  starting_attributes:
    PHY: 15    # 최고 수준의 신체 능력
    INT: 10
    WIL: 13    # 극한 상황 버팀 능력
    AGI: 14
    FDK: 10
    SOC: 6     # 대인 관계보다 임무 중심
  clearance_level: 3
  starting_hp: 35
  starting_san: 44
```

## 6.3 직책 특수 능력

| 능력 | 효과 |
|------|------|
| **분대 전술** | 같은 장면에 MTF 플레이어가 2명 이상 있을 경우, 전투 판정에 +2. 한 명이 공격할 때 다른 한 명이 엄호 |
| **특수 장비 접근** | 챕터 시작 시 재단 창고에서 Level 3 장비 선택 (방호복, 특수 탄약, 전자기 차폐기 등 GM 정의) |
| **외부 임무 권한** | 시설 외부 지역 이동 임무를 단독으로 수행 가능. 다른 직책은 MTF 동행 없이 외부 이동 불가 |
| **명령 거부 저항** | WIL 판정 (DC 14) 성공 시 명백히 비인도적인 명령을 공식 채널로 이의 제기 가능. 직접 거부 시 RP +5 |

## 6.4 MTF의 제약

- 모든 임무는 커맨드의 사전 승인이 필요 (비공개 임무는 FDK 판정 필요)
- 임무 완료 후 보고서 제출 의무. 허위 보고는 RP +5이지만 발각 시 클리어런스 즉각 박탈
- MTF 팀 내 위계가 있다. 선임이 있는 경우 독립 행동에 추가 판정 필요

# 7. 행정직 (Administrative)

## 7.1 직책 개요

행정직은 재단을 움직이는 실질적인 기어다. 예산 관리, 인사 이동, 문서 처리, 정책 입안. 직접 싸우지 않지만 재단 내 가장 광범위한 정보를 가지고 있다.

행정직은 재단 전체를 게임판으로 사용한다.

## 7.2 능력치 배분

```yaml
administrative:
  starting_attributes:
    PHY: 7
    INT: 12
    WIL: 10
    AGI: 8
    FDK: 16    # 재단 구조를 가장 잘 아는 직책
    SOC: 15    # 인맥과 협상이 핵심
  clearance_level: 3
  starting_hp: 19
  starting_san: 35
```

## 7.3 직책 특수 능력

| 능력 | 효과 |
|------|------|
| **문서 처리** | SOC + FDK 판정 (DC 12) 성공 시, 다른 플레이어의 처분 명령·징계 기록을 행정 오류로 처리하여 지연·취소 |
| **인사 조작** | 특정 NPC를 다른 구역으로 발령하여 장면에서 제거 또는 유리한 위치로 이동 (FDK 판정 DC 14) |
| **예산 권한** | 장비, 공간, 이동 수단에 대한 재단 예산을 임의로 배정 가능. 감사 추적이 생긴다 |
| **공식 퇴직 서류** | 행정직 전용. RP 80 이상에서 공식 퇴직 서류를 준비할 수 있다. 성공 시 '명예 퇴직' 엔딩 확정 (FDK + SOC 판정 DC 18) |

## 7.4 행정직의 제약

- 직접 전투 불가 (전투 판정 불가. 회피만 가능)
- 모든 행동이 서류로 추적됨. 은밀 행동은 최고 난이도
- 재단 내부 정치에 항상 노출됨. 상급자 NPC의 의심을 살 수 있음

# 8. 직책 변경 및 승격 규칙

## 8.1 직책 변경 조건

직책 변경은 드물지만 가능하다.

| 변경 경로 | 조건 | 퇴사 포인트 변화 |
|-----------|------|----------------|
| D계급 → 연구원 | D계급 실적 3챕터 인정 + 연구 참여 기록 | -5 (재단이 더 깊이 관여함) |
| 연구원 → 행정직 | FDK 14 이상 + 상급자 추천 | -8 |
| 보안팀 → MTF | PHY 14 이상 + 3번의 격리 위반 대응 성공 | -5 |
| 모든 직책 → 행정직 | SOC 14 이상 + FDK 12 이상 + 공로 기록 | -10 |

## 8.2 강제 직책 변경

플레이어 선택 없이 직책이 변경되는 경우:
- D계급이 처분 명령에서 살아남고 재단이 가치를 인정한 경우 → 강제 연구원 전환
- MTF가 명령 거부로 징계를 받은 경우 → 강제 보안팀 강등
- 모든 직책에서 아메스틱 투여 후 → 기억 소실 상태로 재배치 (새 직책은 GM 결정)

# 9. 직책 간 상호작용 규칙

## 9.1 권한 충돌

| 상황 | 우선순위 |
|------|---------|
| 보안팀이 연구원에게 접근 제한 명령 | 보안팀 우선 (클리어런스 규정) |
| MTF가 D계급에게 임무 지시 | MTF 우선 |
| 행정직이 MTF 임무를 문서상 취소 | 클리어런스 레벨 비교. 동급이면 GM 중재 |
| 연구원이 D계급에게 실험 참여 요구 | 연구원 우선 (D계급은 거부 권한 없음. 단, 플레이어는 RP를 희생하고 거부 가능) |

## 9.2 협력 가능한 행동

직책이 달라도 공동 이익을 위해 협력이 가능하다. 협력 행동은 NPC 재단 직원들이 보기에 수상해 보일 수 있다.

| 협력 유형 | 필요 판정 | 성공 시 효과 |
|-----------|----------|------------|
| D계급 + 연구원: 실험 데이터 공유 | SOC DC 10 | 연구원이 D계급에게 임시 Level 1 정보 제공 |
| 보안팀 + D계급: 탈출 경로 개방 | 보안팀 FDK DC 13 | D계급 RP +4, 보안팀 RP +6 |
| MTF + 연구원: 현장 SCP 분석 | 각각 PHY, INT 판정 | SCP 조사 추리 단서 2배 획득 |
| 행정직 + 전원: 집단 처분 취소 | 행정직 FDK + SOC DC 16 | 해당 챕터 처분 명령 전체 무효화 |

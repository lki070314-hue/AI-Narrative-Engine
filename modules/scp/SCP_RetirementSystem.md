# 1. 퇴사 시스템 개요

## 1.1 정의

**퇴사 포인트 (Retirement Points, RP)** 는 플레이어 캐릭터가 SCP 재단의 통제 구조에서 벗어나는 진전도를 수치화한 것이다.

재단은 직원이 자유 의지로 떠날 수 없는 조직이다. 퇴사는 서류 한 장으로 이뤄지지 않는다. 탈출, 협상, 내부 전복, 변이, 기억 말소 수용 — 모두가 퇴사의 한 형태이며, 각각 다른 경로로 100포인트에 도달한다.

## 1.2 승리 조건

| 조건 | 결과 |
|------|------|
| **RP 100 달성** | 캠페인 엔딩 진입. 퇴사 유형에 따라 6개 분기 중 하나로 진행 |
| **캐릭터 사망** | 해당 캐릭터의 퇴사 시도 종료. 생존 플레이어는 계속 진행 가능 |
| **RP 0으로 강제 감소** | 캐릭터가 재단에 완전히 흡수됨. 이후 NPC화 가능 |

## 1.3 퇴사 포인트는 비밀인가

**플레이어:** 자신의 RP는 알고 있다. 다른 플레이어의 RP는 원칙적으로 모른다. 단, 비밀 미션이나 협력 행동으로 타인의 RP를 알게 되는 경우가 있다.

**GM:** 모든 플레이어의 RP를 실시간으로 알고 있다.

**재단 NPC:** RP 개념 자체를 인식하지 않는다. NPC에게 이것은 "수상한 행동 이력" 또는 "충성도 지수"로 간접적으로 드러난다.

# 2. 퇴사 포인트 획득 조건

## 2.1 일반 획득 조건

| 행동 | 기본 RP |
|------|---------|
| 챕터 클리어 | +5 |
| 개인 미션 완료 | +3 ~ +8 (미션 난이도에 따라) |
| 비밀 미션 완료 | +10 ~ +20 (미션 유형에 따라) |
| 재단 규정 위반 후 생존 | +2 |
| 재단이 금지한 정보에 접근 후 은폐 성공 | +3 |
| 동료 탈출 지원 성공 | +4 |
| 재단 내부 고발 (외부 세력에 정보 전달) | +6 ~ +10 |
| 재단의 처분 명령을 거부 | +5 |
| SCP 개체와 비인가 협력 성공 | +7 |
| 아메스틱 투여 저항 성공 | +3 |

## 2.2 직책별 추가 획득 조건

| 직책 | 특별 획득 조건 | 보너스 RP |
|------|--------------|-----------|
| **D계급** | D계급 처분 명령을 피해 생존 | +4 |
| **D계급** | 연구원이나 보안팀을 역이용 성공 | +3 |
| **연구원** | 금지된 실험 수행 후 데이터 외부 유출 | +8 |
| **연구원** | SCP에 대한 진실을 재단 문서와 다르게 기록 | +5 |
| **보안팀** | 처분 명령을 고의로 지연·실패 | +4 |
| **보안팀** | 다른 플레이어의 탈출 경로를 열어줌 | +6 |
| **기동특무부대** | 임무 보고서에 의도적 허위 기재 | +5 |
| **기동특무부대** | 재단 외부 인물에게 시설 정보 유출 | +8 |
| **행정직** | 퇴직 서류를 합법적으로 처리 성공 | +12 |
| **행정직** | 다른 플레이어의 기록을 말소·조작 성공 | +6 |

## 2.3 협력 보너스

같은 챕터에서 두 명 이상의 플레이어가 공동 탈출·반란 행동에 성공할 경우 추가 RP를 받는다.

| 참여 인원 | 보너스 RP (각각) |
|-----------|----------------|
| 2명 | +2 |
| 3명 | +4 |
| 4명 이상 | +6 |

# 3. 퇴사 포인트 감소 조건

## 3.1 일반 감소 조건

| 상황 | RP 변화 |
|------|---------|
| 재단의 명령에 자발적으로 복종 (거부 가능했음에도) | -2 |
| 비밀 미션을 포기하거나 실패 | -5 |
| 동료 플레이어를 재단에 자발적으로 고발 | -3 (고발한 사람), +5 (고발당한 사람) |
| 아메스틱 자발적 수용 | -10 ~ -20 (말소된 기억 내용에 따라) |
| 재단 충성도 서약 서명 | -8 |
| 재단의 승진 제안 수락 | -5 ~ -15 (직급 상승폭에 따라) |

## 3.2 강제 감소 조건 (플레이어 선택 없음)

| 상황 | RP 변화 |
|------|---------|
| 강제 아메스틱 투여 성공 (저항 실패) | -15 (중요 기억 말소 범위에 따라 최대 -30) |
| SCP 개체에 의한 기억 조작 | GM 결정 (-5 ~ -25) |
| 재단이 캐릭터의 가족·지인을 위협하여 복종 유도 성공 | -10 |

# 4. 퇴사 유형 및 엔딩 분기

RP 100 달성 시 캐릭터의 현재 상태, 선택 이력, 직책에 따라 6가지 엔딩 중 하나에 진입한다.

## 4.1 명예 퇴직 (Honorable Discharge)

**조건:**
- RP 100 달성
- 재단 내에서 공식 승인 경로를 주로 사용 (행정직 주 경로)
- 클리어런스 레벨 4 이상
- 재단에 직접 피해를 준 행동이 3회 이하

**결과:**
재단이 공식 은퇴를 승인한다. 기억소거 없이 민간인 신분으로 전환. 단, 재단 관련 정보는 영구 비밀 유지 조항 서명 하에 보관된다. 감시는 평생 지속되지만 행동 자유는 보장된다.

**분위기:** 가장 안전하지만 가장 덜 자유롭다. 재단의 그늘 안에서의 평화.

## 4.2 자발적 이탈 (AWOL)

**조건:**
- RP 100 달성
- 탈출 관련 행동 이력 5회 이상
- 재단 시설 외부에서 챕터를 마감

**결과:**
재단의 추적을 따돌리고 완전한 도주에 성공. 단, 재단은 추적을 멈추지 않는다. 자유롭지만 쫓기는 삶. 캠페인 종료 후 에필로그로 "5년 후 행방" 서술.

**분위기:** 진정한 자유이지만 영원히 도망자.

## 4.3 기억소거 퇴직 (Amnestic Retirement)

**조건:**
- RP 100 달성
- 아메스틱 관련 이벤트를 복수 경험
- 재단과의 협상 행동 이력

**결과:**
플레이어가 자발적으로 관련 기억의 말소에 동의하는 조건으로 평범한 민간인 신분을 획득. 재단, SCP, 캠페인의 모든 기억이 지워진다. 자유는 얻지만 그 자유를 얻은 이유를 모른다.

**분위기:** 가장 씁쓸한 엔딩. 자유를 얻었으나 자신이 무엇을 잃었는지 모른다.

## 4.4 내부 전복 (Foundation Overthrow)

**조건:**
- RP 100 달성
- FDK(재단지식) 속성 15 이상
- 재단 내부 세력 구축 행동 이력 (동맹 NPC 3인 이상)
- 행정직 또는 연구원 직책 주 경로

**결과:**
재단 내부에서 충분한 영향력을 확보하여 자신이 원하는 방향으로 재단 구조를 변경. O5 위원회 의석 획득, 특정 SCP 격리 해제, 또는 D계급 처우 개선 정책 시행. "퇴사"하지 않고 재단을 바꾼다.

**분위기:** 가장 야망적인 엔딩. 재단은 남지만 그것은 더 이상 예전의 재단이 아니다.

## 4.5 이형화 (Assimilation)

**조건:**
- RP 100 달성
- SCP 개체와의 비인가 접촉 5회 이상
- WIL 10 이하 또는 SAN 30 이하 상태에서 엔딩 진입
- SCP와 비밀 협력 행동 이력

**결과:**
SCP 개체의 영향으로 인간의 범주를 벗어난 존재로 변화. 재단의 규칙이 더 이상 적용되지 않는 존재가 됨. 자유롭지만 인간으로서의 무언가를 잃는다. 에필로그에서 새로운 SCP 개체 항목으로 기록될 수도 있다.

**분위기:** 가장 비인간적인 자유. 승리이자 상실.

## 4.6 소각 처분 회피 (Termination Reversal)

**조건:**
- D계급 전용 엔딩
- 처분 명령이 발동된 상태에서 RP 100 달성
- 탈출 경로 없이 재단 시설 내부에서 달성

**결과:**
처분 직전까지 몰렸으나 재단의 내부 사정 (정보 보유, 협박, 혹은 단순한 생존 능력)으로 처분이 취소되고 신분이 격상. D계급에서 정식 직원으로 전환.

**분위기:** 가장 극적인 반전. 가장 낮은 위치에서 살아남는 것.

# 5. 포인트 추적 규칙

## 5.1 Mission Engine 연계

RP 변화는 Mission Engine의 `rewards.world_effects` 또는 `consequences.on_complete`를 통해 처리된다. 직접 수치 갱신이 아닌 `mission_event`로 발행하고 Director Engine이 서술에 통합한다.

## 5.2 Shadow Engine 연계

Shadow Engine은 모든 플레이어의 현재 RP를 `npc_agendas`로 추적한다. RP가 70 이상이 되면 재단의 감시 수준이 자동으로 올라가는 배경 사건이 예약된다.

| RP 구간 | 재단 감시 수준 |
|---------|--------------|
| 0 ~ 29 | 일반 직원 감시 (루틴 점검) |
| 30 ~ 59 | 요주의 인물 마킹 (수상 행동 보고 의무) |
| 60 ~ 79 | 고위험군 분류 (일정 제한, 이동 추적) |
| 80 ~ 99 | 즉각 처분 대상 또는 아메스틱 투여 대기 |
| 100 | 엔딩 분기 진입 |

## 5.3 플레이어 간 RP 비공개 원칙

다른 플레이어의 RP는 기본 비공개다. 아래 행동으로만 타인의 RP를 알 수 있다.

| 방법 | 알 수 있는 정보 |
|------|---------------|
| 비밀 미션 "동료 감시" 완료 | 대상 플레이어의 RP 정확한 수치 |
| FDK 판정 성공 (DC 15) | 대상 플레이어의 RP 구간 (구체적 수치 아님) |
| 재단 내부 기록 해킹 성공 | 대상 플레이어의 감시 등급 (구간 참조) |

# 7. Mission Engine 연계

## 7.1 RP 갱신 이벤트 포맷

```yaml
# Mission Engine on_complete 또는 on_fail에 포함되는 WorldEffect 확장
retirement_point_event:
  event_type: rp_gain | rp_loss
  target_player_id: string
  delta: int                     # 양수면 획득, 음수면 감소
  reason: string                 # GM 로그용 이유
  is_secret: bool                # true이면 다른 플레이어에게 비공개
```

## 7.2 엔딩 트리거 조건 검증

Mission Engine은 매 챕터 종료 시 모든 플레이어의 RP를 확인한다.

```
if player.retirement_points >= 100:
  → 해당 플레이어의 retirement_type 결정 (§4 조건 검사)
  → Director Engine에 ending_trigger 알림
  → 해당 플레이어는 다음 챕터가 아닌 개인 엔딩 시퀀스로 진입
  → 다른 플레이어들은 캠페인을 계속 진행할 수 있음
```

# 1. 2기 세계 상태 개요

## 1.1 시간 배경

- **1기 챕터 1 종료 후:** 72시간 경과 (3일)
- **현재 시각:** 사이트 내부 표준시 기준 오전 09:14
- **시설 운영 상태:** AMBER-4 (경계 단계. 비상이 아니지만 정상도 아님)

## 1.2 현재 파악된 세계 상태

```yaml
world_state:
  site_status: AMBER-4
  containment_overall: STABLE_WITH_CONCERNS
  player_clearance_access: 현재 브리핑 구역 D-7으로 이동 지시
  recent_events:
    - "관측 의존형 SCP 격리 안정화 완료 (72시간 전)"
    - "SCP 코어 붕괴 확인, 격리 챔버 보수 중"
    - "연구원 3명 전원 생환, 의료 검진 완료"
    - "사이트 전체 AMBER-4 격상 (원인 미공개)"
```

## 2.2 플레이어들이 공식 채널로 알 수 있는 것

- 챕터 1은 "성공"으로 기록됨
- 연구원 3명 생환은 "현장 인원의 적절한 대응" 덕분으로 기술됨 (특정인 명시 없음)
- SCP 코어 붕괴: "평가 중"으로만 처리
- 관련 SCP 항목 번호: [데이터 말소] — Level 3 이상만 열람 가능

# 3. 시설 현황 (공식)

## 3.1 구역 접근 상태

| 구역 | 상태 | 접근 가능 레벨 |
|------|------|--------------|
| 생활 구역 | 정상 | 모든 인원 |
| 행정 구역 | 정상 | Level 1 이상 |
| 연구 구역 A | 제한 | Level 2 이상 (일부 방 봉쇄) |
| 연구 구역 B | 제한 | Level 3 이상 |
| 격리 챔버 섹터 C | **일시 봉쇄** | Level 4 이상 또는 특별 허가 |
| 브리핑 구역 D-7 | **호출 중** | 지정 인원 |
| 지하 2층 이하 | 접근 불가 | 현재 미지정 |

## 3.2 공식 고지된 변경 사항

- 격리 챔버 섹터 C는 "보수 작업"을 이유로 봉쇄
- 사이트 전체 AMBER-4 격상 이유: "예방적 조치"로만 공지
- 다음 D계급 로테이션: 예정일 72시간 연장 (이유 미고지)

# 4. 재단 공식 반응

## 4.1 직속 상관 NPC들의 공개 태도

| NPC 역할 | 공개 태도 | 특이사항 |
|---------|---------|---------|
| 사이트 감독관 | 친절하고 평가하는 시선 | 챕터 1 관련 언급 회피 |
| 직속 부서장 | 평소보다 격식적 | 브리핑 내용 사전 고지 없음 |

# 6. 시작 장면: 호출 방송 직후

## 6.1 Director Engine 서술 가이드

챕터 2는 AMBER-4 방송이 울린 직후부터 시작한다. 아래 서술은 참고 예시이며, Director Engine이 플레이어의 현재 위치와 상황에 맞게 조정한다.

```
[서술 시작 — Director Engine 참고]

방송은 두 번 반복됐다. 그것이 전부였다.

브리핑 구역 D-7. 당신이 이 시설에서 일한 이래로 그 구역이 호출에
사용된 것은 처음이다. D-7은 교육용 브리핑룸이다. 지도와 스크린이 있고,
커피 머신이 있으며, 언제나 약간 서늘하다.

"지정 인원"이 누구인지 방송은 말하지 않았다.
당신 주변의 다른 직원들은 자신이 해당되는지 아닌지를 확인하지 않은 채
자신의 업무를 계속하고 있다.
```

## 6.2 첫 장면 설정

플레이어 행동은 미리 정하지 않는다. Director는 장면을 서술하고, 선택은 플레이어에게 위임한다.

- 방송 직후: 주변 직원들은 각자의 반응을 보이거나 자리를 지키고 있다
- 복도: D-7 방향 표지판이 보인다 (구역 지도상 위치 확인 가능)
- D-7 구역 입구: 보안 스캐너와 담당자가 있다

# 7. 플레이어가 알고 있는 것

## 7.1 공통 인지 사항

- 챕터 1에서 무슨 일이 있었는지 — 자신이 경험한 것
- 연구원 3명이 살아있다
- SCP 코어가 붕괴했다
- 봉투를 열었고, 거기에 단어가 하나 있었다
- 현재 시설은 AMBER-4다

## 7.2 플레이어별 추가 인지

각 플레이어는 자신이 1기에서 어떤 행동을 했는지, 어떤 비밀 미션이 있었는지(있었다면)를 알고 있다. 다른 플레이어의 챕터 1 상세 행동 이력은 서로 알 수도, 모를 수도 있다. 이것은 세션 시작 시 플레이어들이 IC로 결정한다.

## 7.3 플레이어가 모르는 것

- 봉투를 누가 보냈는가
- 방송의 "지정 인원"에 왜 자신이 포함되었는가 (포함되었다고 가정할 경우)
- SCP 코어 붕괴의 실제 의미
- 재단이 챕터 1을 어떻게 평가하고 있는가 (공식 보고서와 실제 평가 간 간극)
- D-7에서 무엇을 들을 것인가
